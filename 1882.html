SOLUTIONS: The C++ standard (Section 6.7) states clearly that a jump into a block must not bypass a declaration statement within the block and such code is therefore ill-formed. A branch to a case label in a switch statement is regarded as a jump in this context. A compiler may or may not detect such errors.

In the example on P174, the value of test that is set is not a determinant of whether the switch statement is ill-formed or not. The code for the switch statement should be according to the language rules in its own right and if there is the possibility for some value of the switch expression that a declaration within the switch block can be bypassed (i.e. the variable declared would be potentially in scope), then the code is ill-formed.

Thus because the declaration for the variable i is bypassed for any switch value and i is in scope to the end of the switch block, this declaration is not allowed.

The declaration for k can be bypassed for switch values other than 1 and is therefore not allowed.

The declaration in the case 1 code section is always legitimate because it appears within a nested block and is therefore not in scope elsewhere in the switch.

While code that cannot be reached may not be strictly illegal, some compilers will flag this as an error when they are able to detect such a situation.
